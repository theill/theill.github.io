Just a quick tip. While upgrading one of my projects from VS.NET 2008 to VS.NET 2010 I ran into a problem I wasn\u2019t able to find a correct solution for it anywhere.<\/p>\n

While building my setup projects I got:<\/p>\n

An error occurred while validating. HRESULT = \u20188001010E\u2019<\/p>\n

Various posts suggested to remove all dependencies but in my case it was caused by an invalid reference to the framework. This is what I did to have it compile again:<\/p>\n

    \n
  • Right-click .vdproj<\/li>\n
  • Select \u201cView\u201d | \u201cLaunch Conditions\u201d<\/li>\n
  • Right-click \u201c.NET Framework\u201d<\/li>\n
  • Select \u201cProperties Window\u201d<\/li>\n
  • Expand \u201cVersion\u201d dropdown<\/li>\n
  • Select correct framework e.g. if it is currently set to \u201c3.5.1234\u201d you might select a similar in the dropdown list i.e. \u201c.NET Framework 3.5\u201d.<\/li>\n<\/ul>\n
